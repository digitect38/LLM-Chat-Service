Predictive Diagnostics
3-Tier Hybrid Knowledge Engine

단일 지식원에 의존하지 않습니다. 데이터 · 경험 · 문서 세 가지 지식 계층을 융합하여 배포 첫날부터 동작하고, 시간이 지날수록 정확해지는 진단을 제공합니다.

Tier 1

Data-Driven

센서 데이터로부터 자동 학습되는 ML 모델. 사람이 놓치는 다변량 패턴을 대규모로 감지합니다.

모델: Isolation Forest, AutoEncoder, LSTM
출력: RUL 예측, 고장 확률, 성능 저하 트렌드
강점: 편향 없음, 대규모 확장 가능
한계: Cold Start 30일+, 해석 어려움
Tier 2

Expert Knowledge

현장 엔지니어가 축적한 조건부 인과 관계. 매뉴얼에 없는, 경험에서만 나오는 지식입니다.

입력: 구조화 템플릿 + 대화 마이닝 + 조치 이력
검증: Tier 1 데이터 기반 자동 사후 검증
스코프: 모델별(기본) 또는 인스턴스별
강점: 조건부 인과, False Alarm 감소
Tier 3

Document-Grounded

매뉴얼 · 표준 문서에서 자동 추출된 진단 지식. 배포 첫날부터 Cold Start 없이 즉시 동작합니다.

추출: Troubleshooting 섹션, 에러코드, PM 절차
표준: SEMI · IEC · ISO 등 산업 표준 자동 추출
강점: Day 1 가용, 공식 절차 기반
한계: 인스턴스 특화 불가, 문서화된 범위만
⚡

Fusion Engine — 다중 순위 진단

세 Tier의 증거를 가중 결합하여 순위화된 진단 결과를 생성합니다. 각 진단에 신뢰도 · 근거 Tier · 매뉴얼 인용이 첨부되어 엔지니어가 최종 판단을 내릴 수 있는 완전한 증거 패키지를 제공합니다.

Use Cases
현장 시나리오

Fab Copilot이 실제 제조 현장에서 어떻게 활용되는지 살펴봅니다.

시나리오 1 — 즉각 대응

야간 근무 중 알 수 없는 에러 발생

비숙련 오퍼레이터가 처음 보는 에러코드를 마주합니다. Chat에 에러코드를 입력하면 매뉴얼 근거와 함께 단계별 조치 방안이 즉시 제공됩니다. 인용 클릭 시 원본 PDF의 해당 위치가 하이라이트됩니다.

📄 Layer 1 RAG → 에러코드 즉시 조회 + 원본 인용
시나리오 2 — 예지 정비

소모품 교체 시점을 놓칠 뻔한 상황

Dashboard에서 Membrane RUL 15% 경고 확인. 3-Tier 진단이 과거 유사 패턴 3건 + 매뉴얼 교체 주기 + 센서 트렌드를 종합하여 "72시간 내 교체 권장"과 함께 부품 번호 및 절차를 제시합니다.

⚡ 3-Tier Fusion → 부품번호 + 절차 + 매뉴얼 인용 통합
시나리오 3 — 지식 축적

베테랑 엔지니어의 노하우 체계화

숙련 엔지니어가 이상 감지 Dashboard에서 "이 원인을 알고 있어요" 클릭. 구조화된 폼에 증상-원인-조치를 입력하면, 시스템이 과거 데이터로 자동 검증(Precision 81%)한 뒤 동일 모델 전체에 규칙을 배포합니다.

🧠 Expert KB → 모델별 자동 배포 + 데이터 검증
시나리오 4 — 다중 도메인 확장

배터리 신규 라인에 동일 플랫폼 적용

반도체 Fab에서 검증된 Fab Copilot 플랫폼에 Battery Domain Pack을 추가합니다. OPC-UA 어댑터 교체, IEC 표준 문서 인덱싱, 배터리 센서 레지스트리 설정만으로 새 라인이 가동됩니다.

🔌 Domain Pack 교체 → 동일 플랫폼, 새로운 도메인
